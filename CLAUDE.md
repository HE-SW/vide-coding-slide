# vide-coding-slide

비개발자 대상 **바이브코딩 강의** (총 4강) 자료 레포. 강의용 스크립트, HTML 슬라이드, 그리고 강의자가 참조할 지식 베이스(`/docs`)를 한 곳에서 관리한다.

## 레포 구조

```
.
├── 1강/
│   └── script.md              # 강의 스크립트 (각 섹션 끝에 [데모: NN-xxx.html] 마커 포함)
│   └── slides/                # HTML 슬라이드 (현재 비어 있음, 작업 예정)
├── 2강/, 3강/, 4강/            # (미정 — 주제 정해지면 추가)
└── docs/                      # 강의자용 지식 베이스
    ├── README.md              # 인덱스
    ├── concepts/              # 큰 개념 (바이브코딩, LLM, Transformer)
    ├── techniques/            # 기법 (Prompt/Context/Harness Engineering, RAG)
    ├── phenomena/             # 현상·한계 (환각)
    ├── people/                # 인물 (Andrej Karpathy)
    ├── tools/                 # 도구·제품 (Claude Code 등)
    └── methodologies/         # 개발 방법론 (워터폴, 애자일, 하이퍼-워터폴)
```

## /docs 작성 규칙

모든 개념 문서는 **3-조건 서사**를 반드시 따른다. 강의 청중이 비개발자이기 때문에 "왜 이렇게 부르나 → 그전엔 무슨 문제였나 → 어떻게 풀렸나" 흐름이 직관 형성에 가장 잘 맞는다.

1. **왜 "{이름}"이라고 부르나** — 인물·연도·어원
2. **그전에는 어떤 문제가 있었나** — 등장 직전의 한계 + 비개발자용 일상 비유 1개
3. **어떻게 해결했나** — 변화 + 1강 스크립트 연결 포인트

추가 섹션:
- **꼬리에 꼬리 (관련 개념)**: 다른 docs 파일로 상호 링크. 새 개념이 나오면 별도 문서로 분리하고 여기에 추가.
- **출처**: 검증된 1차/2차 자료 URL을 하단에 모음. WebSearch로 연도·인물·논문 사실을 직접 확인한 것만 인용.

## 새 카테고리 신설 기준

기본 4개(`concepts/`, `techniques/`, `phenomena/`, `people/`) 안에 안 들어가는 개념이 나오면 새 폴더를 만든다. 예상 후보: `models/` (GPT/BERT/Claude 같은 구체 모델), `methods/` (Few-shot, CoT 같은 세부 기법), `tools/` (Cursor, Claude Code 같은 제품).

## 현재 상태 (2026-05-07 기준)

- **1강 지식 베이스**: 완료. 시드 6개 + 꼬리 3개 = 총 10개 문서.
- **1강 스크립트**: 작성됨 (`1강/script.md`).
- **1강 HTML 슬라이드**: 작업 중. 01–24 중 일부 작성 (스크립트의 `[데모: NN-xxx.html]` 마커가 슬라이드 파일명을 지시).
- **2강 스크립트**: 작성됨 (`2강/script.md`). 주제 = "하니스 세팅 — 이미 박혀 있는 1층 위에 내 프로젝트 2층 한 층 더 올리기". 18개 섹션. 2층 = CLAUDE.md / docs/(PRD·ARCHITECTURE·ADR·UI_GUIDE) / skills/(/harness·/review) / hooks/(caveman-kor 출력토큰 30~50% 절감) 4가지. 클라이맥스 콤보 = 11(흐름) → 12(결과) → 13(claude -p 매커니즘). 마무리 = 15번 "프레임워크 완성 + GitHub CTA". 18번 = 3강 예고 "Dicom Viewer 만들기" 5단계 워크플로우.
- **2강 HTML 슬라이드**: 작성됨 (`2강/slides/01–18 + index.html`). 1강과 동일한 다크 테마, 보라→하늘 그라디언트, Noto Sans KR + JetBrains Mono.
- **2강 HTML 슬라이드**: 미작성. 스크립트 합의 후 진입 예정.
- **3~4강**: 주제 미정. 사용자 지시 대기.

## 작업 시 원칙

- **출처 없이 사실 단정 금지**: 연도·인물·논문 같은 디테일은 반드시 WebSearch로 검증 후 기재. 강의자가 청중에게 잘못된 사실을 전달하면 신뢰가 깨진다.
- **비개발자 톤 유지**: docs는 강의자가 읽지만, 강의자가 그대로 청중에게 옮길 수 있어야 한다. 전문 용어는 한 번 풀어서 설명한 뒤 사용.
- **1강 스크립트 수정은 사용자 영역**: `1강/script.md`는 사용자가 직접 다듬는 파일. AI가 함부로 고치지 않는다.
- **상호 링크 유지**: 한 문서를 추가/수정하면 `docs/README.md` 인덱스와 관련 문서의 "꼬리에 꼬리" 섹션을 함께 갱신.
- **HTML 슬라이드는 별도 작업**: docs 작성과 슬라이드 작성은 분리해서 진행. 사용자가 명시적으로 슬라이드 작업을 요청할 때만 진입.
