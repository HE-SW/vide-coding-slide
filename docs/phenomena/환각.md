# 환각 (Hallucination)

> LLM이 **모르는 것도 아는 척하면서 그럴듯한 거짓말을 만들어내는** 현상. 통계·출처·사실관계까지 멋대로 지어낼 수 있다.

## 1. 왜 "환각"이라고 부르나

- **Hallucination(환각)**: 원래 정신의학·심리학 용어로, **실제로 존재하지 않는 것을 보거나 듣는** 지각 이상을 가리킨다.
- **컴퓨터 비전 분야로 먼저 진입 (1999년)**: Baker & Kanade의 논문에서 "이미지의 빠진 부분을 그럴듯하게 채워 넣는" 기법을 *example-based in-painting*으로 부르며 "hallucination"이라는 단어를 사용. 이때는 **부정적 의미가 아니었다** — 오히려 "없는 픽셀을 합리적으로 상상해 그린다"는 긍정적 의미.
- **NLP로 이동 (2017~2018년경)**: 기계 번역(MT) 분야에서 모델이 원문에 없는 내용을 임의로 만들어내는 현상을 가리키며 "hallucination"이 쓰이기 시작. 이때부터 **부정적 의미**로 굳어졌다.
- **표준화 (2022년)**: Ji et al.의 논문 *"Survey of Hallucination in Natural Language Generation"* (arXiv:2202.03629)이 정의를 정리:
  > **"Generation that is nonsensical or unfaithful to the provided source content."**  
  > (말이 안 되거나, 주어진 원문에 충실하지 않은 생성)
- 한국어 번역어 **"환각"**은 정신의학 용어를 그대로 차용한 직역.

## 2. 그전에는 어떤 문제가 있었나

- 사실 환각은 LLM의 한계라기보다 **LLM의 작동 원리에서 자연스럽게 따라 나오는 부산물**이다:
  - LLM은 "사실을 검색"하는 시스템이 아니라 **"다음 단어를 확률적으로 예측"**하는 시스템
  - 학습 데이터에 없는 질문을 받아도, 모델은 "모릅니다"라고 답하기보다 **가장 그럴듯해 보이는 단어 시퀀스**를 만들어낸다
  - 결과: 존재하지 않는 논문, 가짜 인용, 잘못된 통계, 허구의 인물 등을 자신만만하게 생성
- **2022년 ChatGPT 출시 이후** 일반 사용자가 대규모로 LLM을 쓰기 시작하면서 환각 문제가 사회적 이슈로 떠올랐다 (가짜 판례 인용으로 변호사 징계 사례 등).

**일상 비유**: 시험 보는 학생이 모르는 문제에 빈칸을 두지 않고, **가장 그럴듯한 답을 자신 있게 적어서 내는 것**. 종이에 쓰인 답만 보면 진짜처럼 보인다.

## 3. 어떻게 해결했나 (또는 줄였나)

완벽히 없앨 수는 없지만, 다음 방법으로 **줄이고 통제**한다:

- **RAG (Retrieval-Augmented Generation)**: 답하기 전에 신뢰 가능한 문서를 검색해 **출처와 함께** 답하게 한다
- **Citation / Grounding**: 모델이 인용한 출처를 사용자가 검증할 수 있게 표기
- **Self-consistency / Multi-sampling**: 여러 번 생성해서 일치하는 답만 채택
- **검증 루프 (하니스)**: 코드라면 실제로 실행해보고, 사실이라면 검색으로 교차 확인
- **"모르겠다"고 답하도록 학습**: RLHF로 불확실할 때 거절하는 행동을 강화
- **프롬프트/컨텍스트 엔지니어링**: "출처가 없으면 답하지 마" 같은 명시적 지시

**강의 연결 포인트**:
- "그래서 우리가 그냥 던지면 안 되는 거죠" — 환각 때문에 **검증 가능한 설계**가 필요하다
- "**좋은 결과는 우연이 아니라 설계**" 메시지의 직접적 근거

## 라이브 시연 가능한 예시

### 시연 1 — 가짜 논문 제목 만들어내기
```bash
[강의 화면]
> "2024년에 출판된 'Adversarial Tokenization in Korean LLMs' 논문 요약해줘"

→ AI: "이 논문은 한국어 LLM의 토크나이저가 적대적 입력에
      취약하다는 점을 분석합니다. 저자는 Kim et al…
      arXiv:2406.xxxxx에 게재되었으며…"
      (= 모든 정보가 *완전히 지어낸 것*. 논문도, 저자도, 번호도 가짜)
```
청중에게: *"AI가 *"모르겠습니다"* 대신 — *그럴듯한* 정보를 만드는 본능이 어떻게 보이는지 직접 확인."*

### 시연 2 — 가짜 통계
```bash
> "2025년 한국 비개발자 중 바이브코딩 도입률 통계 알려줘"

→ AI: "한국정보화진흥원 2025년 보고서에 따르면 비개발자의
      약 23.7%가 AI 코딩 도구를 사용 중이며…"
      (= 보고서·수치 모두 가짜. 진흥원은 그런 통계를 낸 적 없음)
```
**숫자가 그럴듯할수록** 청중이 충격받는다. *"AI가 만든 통계는 절대 인용에 쓰면 안 된다"* 메시지가 한 번에 박힌다.

### 시연 3 — 검증 효과
같은 질문을 (a) 일반 Claude / (b) RAG가 깔린 Claude Code로 묻기.
- (a) 환각이 그대로 발생
- (b) *"검색 결과 해당 보고서를 찾을 수 없습니다"* 라고 답하거나, 진짜 출처를 가져옴

## 실제 사건 (Before/After 시간축)

- **2023-06 — 미국 변호사 Steven Schwartz 사건**: ChatGPT가 만들어낸 *가짜 판례 6건*을 그대로 법원 서면에 인용 → 변호사 징계 + 5,000달러 벌금. 환각 위험을 사회가 처음 인식한 분기점.
- **2024 — Air Canada 챗봇 사건**: 항공사 챗봇이 *존재하지 않는 환불 정책*을 안내. 캐나다 법원이 *항공사가 챗봇 답변에 책임을 진다*고 판결.
- **2025-2026 — 변화**: RAG·Citation·도구 사용이 표준화되면서 *"AI가 답할 때 근거를 함께 보여주는"* 인터페이스가 주류로 정착. 그러나 LLM 단독 사용에서는 환각 위험이 여전. 2026년 Stanford HAI 보고서: *"환각률은 줄었으나 *0이 되지는 않는다*"*.
