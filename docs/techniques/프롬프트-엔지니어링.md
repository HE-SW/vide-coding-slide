# 프롬프트 엔지니어링 (Prompt Engineering)

> AI에게 **무엇을 어떻게 요청할 것인가**를 설계하는 기술. "질문을 잘 던지는 능력".

## 1. 왜 "프롬프트 엔지니어링"이라고 부르나

- **Prompt(프롬프트)**: 영어로 "재촉하다·자극하다", 컴퓨팅에서는 사용자가 입력하는 명령줄(`>`)을 가리키던 말. AI 시대에는 **모델에게 주는 입력 텍스트** 전체를 의미하게 됐다.
- **Engineering(엔지니어링)**: 그냥 "물어본다"가 아니라, **반복 가능한 방식으로 결과 품질을 끌어올리는 설계 행위**라는 의미를 강조하기 위해 붙은 단어.
- **언제 등장했나**: 2020년 GPT-3 출시 직후. GPT-3는 "어시스턴트"로 만들어진 모델이 아니었기 때문에, 사용자가 **입력을 어떻게 구성하느냐**에 따라 결과 품질이 극단적으로 달라졌다. 이걸 "트릭처럼 다루는 기술"이라 인식하면서 "prompt engineering"이라는 표현이 자리 잡았다.
- **2022년 ChatGPT 출시 이후** 일반 사용자에게도 알려졌고, "Prompt Engineer"라는 직무까지 등장했다.

## 2. 그전에는 어떤 문제가 있었나

- **GPT-2 이전 (2019년 이전)**: 모델이 충분히 똑똑하지 않아서, 입력을 어떻게 줘도 결과 차이가 크지 않았다. 입력 설계가 의미가 없었다.
- **GPT-3 직후 (2020년)**: 모델은 똑똑해졌지만 **사용 설명서가 없었다**. 같은 질문을 다르게 표현하면 답변 품질이 천차만별이었다. 사람들은 우연히 좋은 답을 받으면 그 표현을 외워서 재사용하는 식으로 대응.
- **결과**: AI를 잘 쓰는 사람과 못 쓰는 사람의 차이가 **운**에 가까웠다. 재현이 어렵고 조직 차원에서 노하우가 쌓이지 않았다.

**일상 비유**: 외국어를 잘 못하는 상대에게 길을 묻는 상황. "Where is the station?"이라고 또박또박 물으면 답을 듣지만, 우물우물 빠르게 말하면 못 알아듣는다. 같은 질문이라도 **표현 방식**이 답을 좌우하던 시기.

## 3. 어떻게 해결했나

- 사용자들이 시행착오 끝에 발견한 핵심 기법들이 **명시적인 방법론**으로 정리됐다:
  - **Zero-shot**: 지시문만 주는 방식 ("이 문장을 영어로 번역해줘")
  - **Few-shot**: 예시 1~3개를 함께 보여주고 패턴을 잡게 하는 방식
  - **Chain-of-Thought (2022)**: "단계별로 생각해줘"를 추가해 추론 정확도를 끌어올림
  - **Role prompting**: "너는 시니어 변호사야"처럼 역할을 부여
- OpenAI·Anthropic·Google이 공식 가이드를 내놓으며 표준화. **재현 가능한 기술**로 자리 잡았다.
- 1강 메시지 연결: **"좋은 결과는 우연이 아니라 설계"**. 프롬프트 엔지니어링은 그 설계의 첫 번째 축.

## 한계와 진화

프롬프트 엔지니어링은 **단발성 질문 품질**에는 강하지만, 실무 LLM 앱(자료 검색·도구 사용·반복 검증)에서는 한계가 있었다. 그래서 2025년부터 **컨텍스트 엔지니어링**이라는 더 큰 개념으로 진화 중. (다음 문서 참조)
