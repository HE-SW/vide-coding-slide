# Role Prompting (역할 프롬프팅 / Persona)

> *"너는 시니어 변호사야"* 한 줄로 답의 톤·범위·전문성을 끌어올리는 패턴. 가장 흔히 쓰이고, 가장 자주 *오용*되는 프롬프트 기법.

> 부모 문서: [프롬프트 엔지니어링](../프롬프트-엔지니어링.md)

## 1. 정의

Role prompting은 모델에게 **특정 역할·페르소나**를 부여해 답의 *전문성·톤·관점*을 그 역할의 것으로 맞추는 기법.

```text
"너는 [역할]이다. 다음 문제를 봐줘: [질문]"
```

## 2. 그전에는 어떤 문제가 있었나

같은 모델이 *맥락 없이* 답하면 — 일반론·중립적 톤으로 묻혀 *전문성*이 안 나옴.

- *"이 코드 봐줘"* → AI: "괜찮아 보입니다." (= 모호)
- *"이 계약서 검토해줘"* → AI: "계약서가 있네요." (= 표면적)

→ 모델 내부의 *전문 지식*을 *끌어올리는 트리거*가 필요했다.

**일상 비유**: 같은 사람이 *변호사 옷을 입었을 때*와 *친구 모드일 때* 답이 다르다. 전문가 옷이 *말투와 디테일*을 바꾼다.

## 3. 어떻게 작동하나

LLM은 학습 데이터에서 *역할별 글쓰기 스타일·지식 수준·관점*을 패턴으로 학습했다. Role prompting은 그 *패턴 영역*을 활성화하는 트리거.

### 효과적인 역할 부여 패턴

```text
[얕음]
"너는 변호사야"

[중간]
"너는 시니어 노동법 변호사야"

[깊음]
"너는 한국 IT 업계 출신, 15년차 노동법 전문 변호사야.
 스타트업 채용 계약서 검토에 특히 강점."
```

→ 구체화할수록 답의 *전문성*과 *맥락 적합성* 향상.

## Before / After

```text
[BEFORE — 역할 없음]
USER: "이 채용 계약서 어때?"
AI:   "전반적으로 무난한 계약서입니다. 몇 가지 확인할 점이..."
       (= 표면적·중립적)

[AFTER — Role prompting]
USER: "너는 한국 IT 업계 노동법 시니어 변호사야.
       다음 채용계약서에서 *직원에게 불리한 조항*을 찾아줘."
AI:   "위험 조항 3건:
       (상) 5조 비밀유지 — 퇴사 후 5년은 과도. 한국 판례상 2년 이내 권장
       (중) 9조 야근수당 — 포괄임금제 명시 누락. 분쟁 시 직원 패소 위험
       (하) 12조 경업금지 — 지역 명시 없음. 한국에서 *집행 불가능* 가능성
       각 조항별 수정안:..."
       (= 전문적·구체적·실행 가능)
```

## 라이브 시연 가능한 예시

```bash
[화면 A — 역할 없음]
> "한국 IT 업계 트렌드 분석해줘"
→ AI: "AI, 클라우드, 모바일이 주요 트렌드..."  (= 일반론)

[화면 B — Role + 페르소나]
> "너는 한국 IT 업계 15년차 시니어 PM이야.
   2026년 한국 IT 업계 트렌드를 *현장 관점*에서 분석해줘.
   특히 비개발자에게 미치는 영향 중심으로."
→ AI: "현장 관점 3대 트렌드:
       1) 바이브코딩 — 비개발자가 직접 *MVP 만드는* 단계
          (예: 영업·마케팅 직군이 직접 대시보드 구축)
       2) 사내 AI 거버넌스 — *어떤 정보를 AI에 줄 수 있나*가 새 KPI
       3) ..."  (= 깊이·맥락·실행 가능)
```

## 4. 실제 사례 (2026)

- **Anthropic의 *Constitutional AI***: 모델 학습 단계에서 *"helpful, harmless, honest"* 라는 *내재 페르소나*를 박음.
- **GPT Custom Instructions**: *"너는 데이터 사이언티스트야"* 같은 페르소나를 GUI로 영구 설정.
- **Claude Projects · ChatGPT Memory**: 사용자가 *역할*을 한 번 설정하면 모든 대화에 자동 적용.
- **Anthropic의 권고 (2025-2026)**: *"역할 부여보다 *행동 지시*가 효과적"* — *"너는 변호사야"* 보다 *"법적 위험을 *심각도별*로 분류해줘"* 가 더 정확. **Role 보다 *Task 명시*가 우선**.

## 한계 — 역할의 함정

### 함정 1 — 역할이 환각을 부른다
```text
USER: "너는 의사야. 이 증상 진단해줘: 두통, 어지러움, 피로."
AI:   "이는 *편두통* 일 가능성이 높습니다."  (= 단정적·환각)
```
의사 페르소나가 *과한 자신감*을 만들어 — 진짜 의사라면 *추가 검사*를 권할 상황에서 단정.

### 함정 2 — 톤만 바뀌고 정확도는 안 변함
연구(2024~2025) 결과 — 역할 부여는 *톤·스타일*에는 효과적이지만 *정확도 자체*는 거의 변하지 않는다는 보고가 다수. *"너는 천재야"* 같은 페르소나는 *플라시보*에 가까움.

### 함정 3 — 사용자도 함께 흘러감
역할 부여로 답이 *그럴듯해 보이면* 사용자가 *검증을 덜* 한다 → [Sycophancy](../../phenomena/Sycophancy.md) 효과까지 결합되면 위험.

## 권장 패턴 (2026)

- **단순 역할 부여 < 행동 지시**: *"너는 변호사야"* 보다 *"법적 위험을 ABC 형식으로 분석해줘"*.
- **단순 역할 < 역할 + 제약**: *"너는 변호사야"* 보다 *"너는 *근거 조문 없이는 단정하지 않는* 변호사야"*.
- **민감 영역(의료·법률)에서는 *항상 disclaimer* 추가**: *"진단이 아니라 일반 정보"* 등을 모델 응답에 포함하도록 강제.
