# v2/2강 — 함께 입력 prompt 운영 노트

> v2 강의(`v2/2강/script.md`, 90분)에서 청중이 자기 PC에 *동시에 입력*하는 prompt **3개**. 1강(6+1회)보다 적은 이유 — 2강은 *강의형*이라 강사 화면 위주, 청중 입력은 *핵심 도구 체험* 3회로 제한.
>
> 각 prompt마다 **위치·예상 출력·강의자 코멘트·fallback**을 묶었다. 강의 직전 Claude Code 버전(v2.1.x / Opus 4.7) 재현 확인 1회 필수 — 명령어 alias나 출력 포맷이 minor 버전마다 변동.

---

## #1 1부 — CLAUDE.md 자동 생성 (`/init`)

**위치**: 슬라이드 10 (`### 10 라이브 — /init으로 CLAUDE.md 만들기`)

```text
> /init
```

**전제 조건**:
- 청중은 자기 PC에서 *빈 폴더 or 본인 토이 프로젝트* 안에서 Claude Code 실행 중
- 폴더가 *완전히 비어있으면* 결과가 빈약 — 강사 화면 공유로 대체

**예상 출력 패턴**:
- Claude가 현재 폴더 분석 (파일 목록·확장자·README 유무)
- `CLAUDE.md` 초안 생성 — 다음 섹션들 자동 추출:
  - *Project Overview* (README 기반)
  - *Tech Stack* (package.json·requirements.txt 등에서)
  - *Build/Test 명령* (scripts 필드에서)
  - *Folder Structure* (실제 폴더 tree)
- 분량은 보통 30~80줄 (200줄 룰 안전권)

**강의자 코멘트**:
- *"내가 직접 적으면 빼먹었을 것"* 을 청중이 찾게 — 토론 5초만
- 폴더가 비어있는 청중 → 강사 화면 보면서 *"실제 프로젝트에서는 이렇게 잡힌다"*
- 200줄 넘으면 → 슬라이드 11(CRITICAL·@import)로 자연스럽게 연결
- *이게 끝이 아님* — 자동 생성된 CLAUDE.md는 *시작점*. 그 위에 *팀 컨벤션·금지 사항*을 사람이 채워 넣어야

**관찰 포인트** (강사 질문):
- Claude가 잡은 *기술 스택*이 정확한가?
- *200줄 넘었나*? 넘었으면 어디를 `@import`로 빼면 좋을까?
- *AI에게 시키면 안 되는 일* 섹션이 자동으로 잡혔나? → 보통 안 잡힘 → *사람이 채워야 한다*는 메시지

**fallback**:
- 폴더 비어있는 청중 — *강사 화면만 보기*
- `/init` 시간이 30초 이상 — 강사 사전 실행 결과 화면 공유
- 출력 너무 김 — *상단 20줄만 같이 읽기*

**소요 시간**: 3분 입력 + 2분 비교

## #2 2부 — 컨텍스트 사용량 시각화 (`/context`)

**위치**: 슬라이드 16 (`### 16 라이브 — /context 같이 보기`)

```text
> /context
```

**전제 조건**:
- 청중 세션이 *비어있으면* 격자가 거의 비어 보임 → 효과 떨어짐
- **사전 작업** (필수): 슬라이드 14·15 진행 전, 모두에게 *1강 회수 한 줄*을 입력하게 시킴:
  ```text
  > 1강에서 본 동심원(프롬프트⊂컨텍스트⊂하니스)을 한 문단으로 정리해줘
  ```
  → 응답 받은 후 슬라이드 16에서 `/context` 실행하면 격자에 색이 차 있음

**예상 출력 패턴**:
- 토큰 사용량 색상 격자 (보통 4~6개 카테고리):
  - System prompt
  - Tools (도구 설명)
  - MCP servers (붙어있으면)
  - Memory files (CLAUDE.md)
  - Messages (대화 내용)
- 총 사용량 N / 200K 같은 표기

**강의자 코멘트**:
- *"시스템 프롬프트가 의외로 크다"* — Anthropic이 박아둔 안전장치·도구 사용법 (1강 시스템 프롬프트 유출 사건과 연결)
- *"MCP를 안 쓰는데 켜져있으면 — 도구 설명만으로 수천 토큰 낭비"* → `/mcp`에서 비활성화 시연
- *"80% 넘었으면 행동"* → /clear · /compact · /rewind 셋 중 선택 판단

**관찰 포인트**:
- *어떤 카테고리가 가장 큰가?* — 청중마다 다름 (MCP 많이 쓰면 그쪽, CLAUDE.md 크면 그쪽)
- *MCP 도구 설명*이 *Messages*보다 큰 경우 → 안 쓰는 MCP 끄기 권장

**fallback**:
- 빈 세션 청중 — 강사 화면이 *충분히 진행된 세션*이면 그걸 보여줌
- `/context` alias가 다른 minor 버전 — `/usage`도 비슷한 정보 (둘 다 시도)

**소요 시간**: 2분 입력 + 3분 격자 해석 (해석에 시간 더 씀)

## #3 3부 — Skill 한 장 만들기 (`skill-creator` 플러그인)

**위치**: 슬라이드 26 (`### 26 라이브 — Skill 한 장 같이 만들기`)

```text
> /install-plugin skill-creator
                    ↓ (설치 후)
> 한국어 README를 자동 생성하는 skill 만들어줘
```

**예상 출력 패턴**:
- skill-creator가 대화형으로 정보 수집:
  - skill 이름 (예: `korean-readme`)
  - description 작성 (트리거 표현 3개 자동 추가)
  - 어떤 도구가 필요한가 (`Read`, `Write` 등)
  - 자동 트리거 vs 수동 호출만
- `.claude/skills/korean-readme/SKILL.md` 자동 생성
- frontmatter + 지시문이 깔린 초안

**강의자 코멘트**:
- *"이게 SKILL.md다"* — 한 장이 자기 PC에 *생기는 순간*을 청중이 직접 봄
- *재사용 단위*가 무엇인지 손에 잡힘 — 매번 *"한국어 README 만들어줘"* 입력하지 않고 `/korean-readme` 한 줄
- 청중에게 *"여러분이 매일 반복하는 작업 중 — 한 장으로 묶을 수 있는 게 있나?"* 한 박자 질문

**관찰 포인트**:
- description에 *사용자가 쓸 법한 표현 3개* 가 들어갔나? → Claude의 *자동 트리거*에 결정적
- *Frontmatter*의 `allowed-tools` 가 *최소 권한*으로 잡혔나? → 보안 좋은 패턴

**fallback** (시간 부족 또는 플러그인 설치 실패):
- 강사 사전 작성한 SKILL.md를 화면 공유 — *프롬프트 vs Skills 차이*만 메시지로 전달
- skill-creator 없으면 — `mkdir -p .claude/skills/korean-readme` 후 SKILL.md 수동 작성 (시간 5배)

**소요 시간**: 5분 (대화형이라 가장 김 — 3부 20분 중 1/4)

## 강의 직전 재현 체크리스트

발표 24시간 전에 *반드시 1회 재현*:

- [ ] `/init` — 빈 폴더와 실제 프로젝트 폴더 각각 1회씩 (출력 분량·구조 확인)
- [ ] `/context` — 슬라이드 14·15 진행 분량 시뮬레이션 후 격자 확인
- [ ] `/install-plugin skill-creator` — 설치 시간 측정 (네트워크 환경 따라 변동)
- [ ] Claude Code 버전 `claude doctor`로 v2.1.x 확인
- [ ] 모델 `/model`로 Opus 4.7 또는 Sonnet 4.6 확인

각 단계 *어디서 막히면 어떻게 우회*할지 fallback 1줄씩 머릿속에 — 라이브에서 *멈춤 = 청중 이탈*.
