# army/2강 — 함께 입력 prompt 묶음

> 강의자 운영 메모: 2강에서 청중이 *Before/After 비교*를 자기 PC에서 직접 보는 prompt 12개.
> 1강과 달리 *대조 시연*이 핵심 — 같은 prompt를 *조건만 바꿔서* 두 번 시킴.

---

## 사전 준비물 (강의 자료로 미리 배포)

청중에게 *zip 파일 1개*를 미리 배포. 안에:
- `before/` — 빈 폴더
- `after/` — CLAUDE.md, UI_GUIDE.md가 깔린 폴더
- `samples/` — 회사 슬로건 예시 3개, 환불 정책 PDF, 회의록 텍스트 1개

---

## #1 도입 — Before/After 충격 (Pomodoro 다시)

### Before
```text
[before/ 폴더에서]
> Pomodoro 타이머 만들어줘. 다크 테마, 미니멀.
```

### After
```text
[after/ 폴더에서, 같은 prompt]
> Pomodoro 타이머 만들어줘. 다크 테마, 미니멀.
```

**예상 차이**:
- Before — 1강과 마찬가지로 청중마다 다 다름
- After — 폴더 구조 / 색깔 / 폰트가 *거의 모두 비슷한 결과*로 모임

**강의자 코멘트**:
- *결과가 모이는* 이유 = `CLAUDE.md` + `UI_GUIDE.md` 가 *컨텍스트로 자동 주입*되기 때문
- 같은 AI에 같은 prompt지만, *받는 컨텍스트가 다르면 다른 답*

**소요**: 10분 (Before 5분 + After 5분)

---

## #2~#3 프롬프트 엔지니어링 — Zero/Few-shot

### #2 Zero-shot
```text
> 우리 회사 슬로건 5개 만들어줘.
  업종은 의료 AI 솔루션이야.
```

### #3 Few-shot
```text
> 우리 회사 슬로건 5개 만들어줘.
  업종은 의료 AI 솔루션이야.

  좋은 슬로건 예시 (다른 회사):
  - Nike: "Just Do It"
  - Apple: "Think Different"
  - Tesla: "Accelerate the World's Transition to Sustainable Energy"

  위 예시처럼 짧고 강한 동사형 슬로건으로 부탁해.
```

**예상 차이**:
- Zero-shot — 길고 모호한 카피 (*"환자의 미래를 위한 정밀한 의료 AI"* 류)
- Few-shot — Nike/Apple 톤 학습된 짧은 동사형 (*"Heal Faster"*, *"Decode the Body"* 류)

**강의자 코멘트**:
- *"예시 3개만 넣어도 답이 이렇게 바뀝니다"* → 프롬프트 엔지니어링의 위력
- 단, 예시는 *비슷한 도메인*이 아니어도 됨. *형식 학습*이 핵심

---

## #4 Chain-of-Thought 시연

```text
> 우리 부서 9명이 회식 가는데
  3명은 해산물 못 먹고, 2명은 술 안 마시고, 1명은 비건이야.
  강남역 근처 추천 식당 3곳. 단계별로 생각하고 답해.
```

**예상 출력 패턴**:
- *"단계별로 생각하고 답해"* 한 줄로 풀이 과정이 따라 나옴
- 1) 제약 조건 정리 → 2) 후보 식당 검토 → 3) 최종 추천 3곳
- 추천이 *제약을 어김없이 만족*하는 경향이 강해짐

**강의자 코멘트**:
- 일상 비유: *수학 시험에서 풀이 과정 적기*. 답만 적으면 틀리기 쉬운데 과정을 적으면 정확

---

## #5~#6 컨텍스트 엔지니어링 — RAG 시험장 비유

### #5 컨텍스트 없음
```text
> 우리 회사 환불 정책에 따라
  이 고객 메일에 답장 써줘.

  [고객 메일 내용 붙여넣기]
```

### #6 컨텍스트 첨부
```text
> 우리 회사 환불 정책은 다음과 같아:

  [환불 정책 본문 붙여넣기 또는 PDF 첨부]

  이 정책에 따라 아래 고객 메일에 답장 써줘.

  [고객 메일 내용]
```

**예상 차이**:
- #5 — *환각해서* 없는 정책을 만들어 답함. *"30일 이내 무조건 환불"* 같은 *우리 회사가 안 하는* 정책
- #6 — 정책에 정확히 맞는 답장. *"교환만 가능, 환불 불가"* 같은 *진짜 정책*

**강의자 코멘트**:
- 일상 비유: *시험장에 책 들고 가기*
- RAG = 시험장 책 자동 검색 시스템

---

## #7 CLAUDE.md 깔기

```text
[빈 폴더에서]
> /init
```

**예상 출력**:
- 폴더 내용을 자동으로 읽고 (빈 폴더면 *"이 프로젝트는 무엇인가요?"* 추가 질문)
- CLAUDE.md 초안 자동 생성

**강의자 코멘트**: 청중에게 *자기 부서 작업 규칙* 한두 줄을 CLAUDE.md에 추가하라고 요청. 예시 제공:

```markdown
## CRITICAL
- 모든 응답은 한국어로
- 외부 고객 메일은 정중체
- 회의록은 Action Item 가장 위에
```

---

## #8 CLAUDE.md 효과 확인

```text
[CLAUDE.md 깐 폴더에서]
> 다음 회의록을 우리 규칙대로 요약해줘.

  [회의록 텍스트 — samples/에 배포된 것 사용]
```

**예상 출력 패턴**:
- Action Item이 가장 위로 올라옴
- 정중체 변환됨
- 한국어로만 답변

**강의자 코멘트**:
- 같은 prompt를 *CLAUDE.md 없는 폴더*에서 시키면 *Action Item 위치 / 톤*이 매번 다름
- *프로젝트 헌법* 한 장으로 결과가 모입니다

---

## #9 Skills — 슬래시 명령

```text
> /init
> /review
```

**예상 출력**:
- `/init` — 프로젝트 초기화 (위 #7과 동일)
- `/review` — 현재 폴더 코드를 자동 검토. *"이 부분이 환각 위험"*, *"이 함수는 테스트 미커버"* 식의 리포트

**강의자 코멘트**:
- 강의자 본인 환경에서 만든 *커스텀 skill*도 같은 식으로 작동 시연

---

## #10 Hooks — 끄고/켜고 비교

### Hook 끔
```text
[설정에서 hooks disable 후]
> 다음 영어 문서를 한국어로 요약해줘.

  [영어 문서 1장]
```

### Hook 켬
```text
[hooks enable 후, 같은 prompt]
> 다음 영어 문서를 한국어로 요약해줘.

  [같은 영어 문서]
```

**예상 차이**:
- Hook 끔 — *"이 paper는 transformer architecture를 활용한 multi-modal learning approach…"* 식으로 영어 단어 다수 섞임
- Hook 켬 — *"이 논문은 트랜스포머 구조를 활용한 다중 양식 학습 방식…"* 자연스러운 한국어
- 토큰 사용량 — Hook 켬 쪽이 *30~50% 적음* (caveman-kor 사례 기준)

**강의자 코멘트**:
- 일상 비유: *맞춤법 검사기*

---

## #11 MCP — Figma 시연

```text
[Figma MCP 연결된 강의자 환경에서]
> 이 Figma 디자인 (URL 첨부) 을 React 코드로 만들어줘.
```

**예상 출력 패턴**:
- Claude가 Figma MCP 도구를 *직접 호출*
- 디자인 토큰 (색상·간격·폰트) 추출
- React + Tailwind 코드 출력
- 컴포넌트 매핑 제안 (*"Button은 shadcn/ui Button 사용 추천"* 식)

**강의자 코멘트**:
- 비개발자에게 가장 *시각적으로 인상적*인 시연
- Figma는 *디자이너 영역*인데 코드까지 *비개발자가 직접 받아내는* 모먼트

**fallback**: Figma MCP 설정 없으면 → GitHub MCP, Drive MCP 등 다른 MCP로 대체

## #12 서브에이전트 — 병렬 분업

```text
> 우리 회사 1년치 보고서 50개를 다음 3가지 관점에서 분석해줘:
  1. 매출 트렌드
  2. 고객 만족도 변화
  3. 신규 사업 진척도

  각 관점은 별도 서브에이전트에게 위임해서 병렬로 진행해.
```

**예상 출력 패턴**:
- Claude가 자동으로 3개 서브에이전트 띄움
- *"매출 분석 에이전트 시작"*, *"만족도 분석 에이전트 시작"* 등 진행 로그
- 약 5~10분 후 합본 리포트

**강의자 코멘트**:
- 한 AI가 50개를 순차로 읽으면 *Lost in the Middle* (1강 단어 떠올리기)
- 작은 맥락에서 일하는 *서브에이전트*는 정확도가 더 높음

**fallback**: 보고서 50개 없으면 → 강의 자료로 *샘플 보고서 5개* 사전 배포해 시연
