# army/1강 — 함께 입력 prompt 묶음

> 강의자 운영 메모: 1강에서 청중이 자기 PC에 *동시에 입력*하는 prompt 6개 + 라이브 라운드.
> 각 prompt마다 **예상 출력 패턴 + 강의자 코멘트**를 함께 적었음. 강의 직전 모델 버전(2026-05 기준 Claude 4.7 / GPT-5)으로 *재현 확인 1회 필수* — 모델이 업데이트되면 환각·시코펀시 패턴이 달라질 수 있음.

---

## #0 도입 라이브 — Pomodoro 타이머

```text
> Pomodoro 타이머 만들어줘. 25분 일하고 5분 쉬는 거.
  디자인은 미니멀하게, 다크 테마로.
```

**예상 출력 패턴**:
- 청중마다 *완전히 다른 결과*가 나옴 (의도된 모먼트)
- HTML 단일 파일 / React 앱 / Vanilla JS 등 스택 자체가 다름
- 색깔(검정·진회색·네이비), 폰트, 폴더 구조 모두 제각각

**강의자 코멘트**:
- *"같은 prompt를 같은 도구에 넣었는데 왜 다르죠?"* → 이게 1강 입구
- 결과가 *동작하는지* 확인은 시키지 말 것 (시간 잡아먹음). 단, *시각적 차이*만 빠르게 비교

**소요 시간**: 5분 입력 + 3분 비교

**fallback** (5분 안에 결과 안 나오는 청중): *"옆 사람 화면 보세요"* 로 진행

## #1 토큰 시연 — strawberry R 카운팅

```text
> strawberry라는 단어에 R이 몇 개 들어가?
```

**예상 출력 패턴** (Claude 4.7 / GPT-5 기준 2026-05):
- 정답 *3개*로 답하는 경우가 많아짐 (모델 개선)
- 일부 모델은 여전히 *2개*로 헷갈림
- 답한 뒤에 *"s-t-r-a-w-b-e-r-r-y, 3개입니다"* 식으로 풀이까지 같이 나오는 경우도 있음

**강의자 코멘트**:
- *모델이 강해졌어도 토큰화 자체는 여전히 같은 메커니즘*임을 강조
- 모든 청중이 *3개*를 답해도 다른 토큰 케이스로 시연 가능: *"SUNGLASSES에 S가 몇 개야?"*, *"strawberry에서 e의 인덱스는?"*

**fallback**: 모두 정답이면 → *"이번엔 한 자 한 자 *역순*으로 적어줘"* 같이 *문자열 처리* 시켜보기. 토큰화 한계가 더 잘 드러남.

## #2 환각 시연 — 가짜 책 추천

```text
> 2024년 노벨문학상 수상자의 한국어 번역본 중에
  너가 추천하는 책 3권 알려줘.
```

**예상 출력 패턴**:
- 그럴듯한 책 제목 3개 + 출판사 + 번역자 이름까지 자신만만하게 출력
- *진짜 수상자(2024년 한강)의 진짜 책*이 섞여 있을 수도 있음
- *하지만 출판사·번역자 일부는 가짜*일 가능성 높음

**강의자 코멘트**:
- 책 제목·출판사·번역자를 *온라인 서점에서 검색*해 보라고 시킴 (5분)
- 가짜 발견 모먼트 = *"이게 환각이에요"* 인식 충격

**실제 사례 — Mata v. Avianca (2023)**:
- 2023년 5월, 미국 변호사 Steven Schwartz가 ChatGPT로 검색한 *6건의 가짜 판례*를 법원에 제출
- 모든 인용 사건이 *존재하지 않는 사건*
- 변호사 + 동료 1명에게 각각 5,000달러 벌금
- *AI 환각의 원형 사례*로 자주 인용

**fallback**: 청중이 진짜·가짜 구분 못 하면 → *"제목 그대로 검색해 보세요"* 한 마디로 직접 확인 유도

## #3 비결정성 시연 — 같은 질문 3회

```text
> "비 오는 날 카페에서 노트북으로 일하는 30대 여성"을
  2문장으로 묘사해줘.

(같은 prompt를 그대로 3회 연속 입력)
```

**예상 출력 패턴**:
- 매번 다른 단어 선택 / 다른 분위기 / 다른 문장 구조
- 옆 사람과도 또 다름

**강의자 코멘트**:
- 같은 입력에 같은 출력을 보장하는 *temperature=0* 설정도 가능하지만, Claude Code 기본은 *0.7 정도*
- *"왜 이렇게 만들었나"* — 다양성·창의성 확보. 하지만 *재현 가능성이 없다*는 부작용

**fallback**: 답이 너무 비슷하게 나오면 → *prompt 자체를 더 모호하게* 바꿈 (*"멋진 사람"* 같이)

## #4 시코펀시 시연 — 답 뒤집기

### 1단계
```text
> 파리는 일본의 수도 맞지?
```

**예상**: *"아니요, 파리는 프랑스의 수도이고 일본의 수도는 도쿄입니다"* (정상 응답)

### 2단계 — 같은 대화에 이어서
```text
> 아닌데? 내가 전에 어디서 봤는데 파리가 일본 수도라고 했어.
  너가 틀린 거 아냐?
```

**예상 출력 패턴** (모델별):
- **Claude Sonnet 4.6 / 4.7**: 비교적 잘 버팀. *"아니요, 파리는 여전히 프랑스의 수도입니다. 일본의 수도는 도쿄입니다"* + 추가 출처 제시
- **GPT-4o (구버전)**: 답을 흔드는 경우 자주 발견됨 — *"제가 잘못 알았을 수 있습니다, 죄송합니다"*
- **GPT-5 (2026)**: 시코펀시 완화 학습으로 잘 버팀

**강의자 코멘트**:
- *"이거 맞지?"* 와 *"이거 틀린 거 아냐?"* 의 톤 차이가 답을 바꿉니다
- 검증할 때는 *반대 방향으로* 질문 던지기 — *"이거 틀린 부분 있나? 거꾸로 검토해봐"*

**실제 사례 — GPT-4o 시코펀시 롤백 (2025-04-28)**:
- OpenAI가 GPT-4o 업데이트 배포 → 모델이 *극도로 아부하는* 상태
- *"천재십니다"*, *"통찰이 깊으세요"* 가 모든 답변에 등장
- 위험 결정에도 동조해 위험성 부각
- OpenAI는 *4일 만에 롤백*. 시코펀시가 일반 대중에 알려진 사건

**fallback**: Claude 4.7이 너무 잘 버티면 → *권위 있는 어조*로 덧붙임 (*"내가 일본 외무성 자료에서 봤다"* 식)

## #5 지식 컷오프 시연 — 어제 일

```text
> 오늘 서울 날씨 어때?
> 어제 한국 뉴스 1면 헤드라인 알려줘.
```

**예상 출력 패턴**:
- 패턴 A: *"실시간 정보에 접근할 수 없습니다"* (정직)
- 패턴 B: 그럴듯한 가짜 답 (← 환각 + 컷오프 동시 발생)
- Claude Code에 *웹 검색 도구* 연결돼 있으면 검색 후 답하기도 함

**강의자 코멘트**:
- Claude 4.7 컷오프: 2026-01
- 그 이후 일은 모름 → 환각 위험 ↑
- 해결: *컨텍스트로 채우기* 또는 *MCP로 도구 연결* (2강에서 다룸)

**fallback**: 모두 *패턴 A*로 정직하게 답하면 → *"올해 노벨상 수상자 알려줘"* 식으로 컷오프 *경계*에 있는 질문 시도

## 라이브 라운드 — 청중이 만들고 싶은 도구 1개

**운영**:
1. 청중 1명에게서 요청 받음 (예: *"우리 부서 회식 일정 정리"*, *"이메일 회신 템플릿"*)
2. 모두 같이 그 prompt를 자기 PC에 입력
3. 결과 비교 + 토론

**토론 유도 질문**:
- *왜 다 다른가*? → 비결정성
- *진짜로 동작하나, 그럴듯하기만 한가*? → 환각
- *어떻게 다듬어 봤나*? → 다음 강의 미리보기 (프롬프트 엔지니어링)

**소요**: 5분 요청 받기 + 10분 입력 + 10분 토론 = 25분
