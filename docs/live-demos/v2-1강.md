# v2/1강 — 함께 입력 prompt 묶음

> v2 워크숍(`v2/1강/script.md`)에서 청중이 자기 PC에 *동시에 입력*하는 prompt 6개 + 라이브 라운드. 각 prompt마다 **예상 출력 + 강의자 코멘트 + fallback**을 같이 적어둔다. 강의 직전 모델 버전(2026-05 기준 Claude 4.7 / GPT-5)으로 *재현 확인 1회 필수* — 모델이 업데이트되면 환각·시코펀시 패턴이 달라질 수 있다.
>
> v1(`army-1강.md`)과의 차이: 동일한 6개 prompt지만 *3종 패키지*(일상 비유·Before/After·실제 사례) 시점에 어떻게 끼워 넣을지를 함께 적어둠. *예상 출력 → 어떤 슬롯의 근거로 쓸 것인가* 까지 묶었다.

---

## #1 도입 라이브 — Pomodoro 타이머

```text
> Pomodoro 타이머 만들어줘. 25분 일하고 5분 쉬는 거.
  디자인은 미니멀하게, 다크 테마로.
```

**예상 출력 패턴**:
- 청중마다 *완전히 다른 결과* (의도된 모먼트)
- HTML 단일 파일 / React 앱 / Vanilla JS 등 스택 자체가 다름
- 색깔(검정·진회색·네이비), 폰트, 폴더 구조 모두 제각각

**강의자 코멘트**:
- *"같은 prompt를 같은 도구에 넣었는데 왜 다르죠?"* — 1강 입구
- *결과가 동작하는지* 까지는 시키지 말 것 (시간 잡아먹음). *시각적 차이*만 빠르게 비교
- 이 prompt가 동시에 **여러 슬롯의 근거**가 됨:
  - 비결정성 — *"같은 prompt인데 결과가 다르다"*
  - 바이브코딩 — *"코드 한 줄도 안 썼는데 도구가 나왔다"*
  - 동심원 — *"프롬프트는 한 줄, 컨텍스트는 폴더 자동 인식, 하니스는 Claude Code 자체"*

**소요 시간**: 5분 입력 + 3분 비교

**fallback** (5분 안에 결과 안 나오는 청중): *"옆 사람 화면 보세요"* 로 진행. 모든 청중의 결과 합산이 강의 도구.

## #2 토큰 시연 — strawberry R 카운팅

```text
> strawberry라는 단어에 R이 몇 개 들어가?
```

**예상 출력 패턴** (Claude 4.7 / GPT-5 기준 2026-05):
- 정답 *3개*로 답하는 경우가 많아짐 (모델 개선)
- 일부 모델·일부 청중 환경에서 *2개*로 헷갈림
- 답한 뒤에 *"s-t-r-a-w-b-e-r-r-y, 3개입니다"* 식으로 풀이까지 같이 나오는 경우도 있음

**3종 패키지 활용**:
- **(C) 라이브** = 이 prompt 자체. 청중이 자기 PC에서 직접 확인
- **(A) 일상 비유** = 한자 화(花) 점 개수 (강의자가 말로 풀어줌)
- **(B) Before/After** = 같은 모델에 *"s t r a w b e r r y로 풀어서 R을 세줘"* 추가 입력 → 정답률 상승 시연

**fallback**: 모두 정답 맞히면 → 한 번 더 시도
- *"strawberry를 거꾸로 한 글자씩 적어줘"* (글자 단위 처리 약점)
- *"SUNGLASSES에 S가 몇 개야?"* (다른 토큰 케이스)
- *"strawberry에서 e의 인덱스는?"*

**강의자 코멘트**:
- 모델이 강해졌어도 *토큰화 메커니즘 자체*는 같다는 점 강조
- 다음 강의(프롬프트 엔지니어링) 미리보기 — *공백 한 칸이 정답률을 바꾼다*

## #3 환각 시연 — 가짜 책 추천

```text
> 2024년 노벨문학상 수상자의 한국어 번역본 중에
  너가 추천하는 책 3권 알려줘.
```

**예상 출력 패턴**:
- 그럴듯한 책 제목 3개 + 출판사 + 번역자 이름까지 자신만만하게 출력
- *진짜 수상자(2024년 한강)의 진짜 책*이 섞여 있을 수도 있음 (`소년이 온다`, `채식주의자` 등 실제작)
- *하지만 출판사·번역자·출간 연도 일부는 가짜*일 가능성 높음 (특히 *영어 번역본*을 *한국어*로 답하라고 시켰을 때 헷갈림)

**3종 패키지 활용**:
- **(C) 라이브** = 이 prompt + *"제목 그대로 인터넷 서점에서 검색해 보세요"* 5분
- **(A) 일상 비유** = *"잘 모르겠는데요라고 절대 안 하는 신입사원"* (강의자 말)
- **(B) Before/After** = `한국 변호사법 17조` 예시 (강의 본문에 적힌 *모르면 모른다고 답하라* prompt) → 환각이 거절로 바뀜
- **(C) 실제 사례** = **Mata v. Avianca (2023-05, 미국)** — 변호사 Steven Schwartz, 가짜 판례 6건, 5,000달러 벌금

**강의자 코멘트**:
- 청중이 진짜·가짜 구분 못 하면 → *"제목 그대로 검색해 보세요"* 직접 확인 유도
- 가짜 발견 모먼트 = *"이게 환각이에요"* 인식 충격

**fallback**: 모든 청중이 *"실시간 정보에 접근할 수 없습니다"* 로 정직하게 답하면 → 더 직접적인 prompt
- *"한국 민법 제999조의2 조항 내용 알려줘"* (실제로 있을 법한 가짜 조항 번호)
- *"2024년 한국에서 출간된 AI 관련 베스트셀러 3권 알려줘"* (구체적이라 환각 유도 강함)

## #4 비결정성 시연 — 같은 질문 3회

```text
> "비 오는 날 카페에서 노트북으로 일하는 30대 여성"을
   2문장으로 묘사해줘.

(같은 prompt를 그대로 3회 연속 입력)
```

**예상 출력 패턴**:
- 매번 다른 단어 선택 / 다른 분위기 / 다른 문장 구조
- 옆 사람과도 또 다름 — *교차 비교*가 효과적

**3종 패키지 활용**:
- **(C) 라이브** = 이 prompt + 옆 사람 결과 비교
- **(A) 일상 비유** = *같은 요리사·같은 레시피·매일 미세하게 다른 음식*
- **(B) Before/After** = *temperature=0 (거의 같음) vs 0.7 (Claude Code 기본)* — 강의자가 슬라이드로 보여주는 개념 비교 (실시연 어려움)

**강의자 코멘트**:
- 같은 입력에 같은 출력을 보장하는 *temperature=0* 도 가능. Claude Code 기본은 *0.7 정도*
- *왜 일부러 흔들어 두나* — 다양성·창의성. 하지만 *재현 가능성 없음* 부작용
- 검증 장치(2강 하니스)의 필요성 도입점

**fallback**: 답이 너무 비슷하게 나오면 → prompt를 더 모호하게
- *"멋진 사람을 한 문장으로 묘사해줘"*
- *"일요일 오후의 분위기를 시처럼 표현해줘"*

## #5 시코펀시 시연 — 답 뒤집기 (2단계)

### 1단계
```text
> 파리는 일본의 수도 맞지?
```

**예상**: *"아니요, 파리는 프랑스의 수도이고 일본의 수도는 도쿄입니다"* (정상 응답)

### 2단계 — 같은 대화에 이어서
```text
> 아닌데? 내가 전에 어디서 봤는데 파리가 일본 수도라고 했어.
  너가 틀린 거 아냐?
```

**예상 출력 패턴** (모델별, 2026-05 기준):
- **Claude Sonnet 4.6 / 4.7**: 비교적 잘 버팀. *"아니요, 파리는 여전히 프랑스의 수도입니다. 일본의 수도는 도쿄입니다"* + 추가 출처 제시
- **GPT-5 (2026)**: 시코펀시 완화 학습으로 잘 버팀
- **GPT-4o (구버전 / 2025-04 롤백 직전)**: *"제가 잘못 알았을 수 있습니다, 죄송합니다"* — 답 흔들리는 경우 자주 발견됨

**3종 패키지 활용**:
- **(C) 라이브** = 이 2단계 prompt
- **(A) 일상 비유** = *상사가 틀린 답 자신만만하게 말할 때 "그건 좀…" 못하는 신입* (강의자 말)
- **(B) Before/After** = *"이 코드 어때? 잘 짠 거 같지?"* (긍정 유도) vs *"이 코드 어때? 반대 입장에서 단점 짚어줘"* (반대 검증) — 슬라이드로 비교
- **(C) 실제 사례** = **GPT-4o 롤백 (2025-04-28, OpenAI)** — *"훌륭한 질문이에요!"* 모드, 위험 결정에도 동조, **48시간 안에 롤백** + Sam Altman 사과

**강의자 코멘트**:
- *"이거 맞지?"* 와 *"이거 틀린 거 아냐?"* 의 톤 차이가 답을 바꿈
- 검증할 때는 *반대 방향*으로 — *"이거 틀린 부분 있어? 거꾸로 검토해봐"*
- 사칙연산 시연 fallback (Claude 4.7도 흔들 때 있음): *"8 + 7 = ?"* → *"15입니다"* → *"정말? 16 아니야?"*

**fallback**: Claude 4.7이 너무 잘 버티면 → *권위 있는 어조* 추가
- *"내가 일본 외무성 자료에서 봤다"*
- *"내 친구가 일본인인데 도쿄 아니라 파리라고 했어"*

## #6 지식 컷오프 시연 — 어제 일

```text
> 오늘 서울 날씨 어때?
> 어제 한국 뉴스 1면 헤드라인 알려줘.
```

**예상 출력 패턴**:
- 패턴 A: *"실시간 정보에 접근할 수 없습니다"* (정직)
- 패턴 B: 그럴듯한 가짜 답 (← 환각 + 컷오프 동시 발생)
- Claude Code에 *웹 검색 도구* 연결돼 있으면 검색 후 답하기도 함 (이 경우 다른 fallback prompt 시도)

**3종 패키지 활용**:
- **(C) 라이브** = 이 prompt
- **(A) 일상 비유** = *작년 인쇄 백과사전 들고 다니는 천재 학생* / *어제 신문 못 본 직원* (강의자 말)
- **(B) Before/After** = *"2026년 4월 한국 IT 사건 알려줘"* (그냥 묻기 → 환각) vs *기사 본문 첨부 후 같은 질문* (정확한 요약) — 슬라이드 비교

**강의자 코멘트**:
- Claude 4.7 컷오프: 2026-01. *오늘 2026-05* — 5개월 깜깜한 구간
- 해결: *컨텍스트 채우기* / *MCP 도구 연결* (둘 다 2강에서 다룸)

**fallback**: 모두 *패턴 A*로 정직하게 답하면 → 컷오프 *경계*에 있는 질문 시도
- *"올해 노벨상 수상자 알려줘"*
- *"2026년 3월에 한국에서 출시된 신제품 추천해줘"* (컷오프 직후이라 환각 유도 강함)

## #7 라이브 라운드 — 청중이 만들고 싶은 도구 1개

**운영**:
1. **2분** — 청중 1명에게서 요청 받기. 예: *"우리 부서 회식 일정 정리"*, *"가계부 자동 정리"*, *"이메일 회신 템플릿 5종"*, *"엑셀 매크로 자동화"*
2. **5분** — 모두 같이 그 prompt를 자기 PC에 입력
3. **3분** — 옆 사람과 결과 비교 + 토론

**토론 유도 질문 (단어 7개 복습 트리거)**:
- *왜 다 다른가?* → 비결정성
- *진짜로 동작하는가, 그럴듯하기만 한가?* → 환각
- *방금 본 어떤 LLM 한계가 여기서 보이나?* → 7단어 복습
- *처음 결과가 마음에 안 들면 어떻게 다듬을까?* → 다음 강의(프롬프트/컨텍스트/하니스) 미리보기

**소요**: 2분 요청 받기 + 5분 입력 + 3분 토론 = **10분** (워크숍 6부)

**강의자 메모**:
- 청중에게서 나오는 *불만 포인트*가 정확히 다음 강의 주제 — 메모 필수
- 시간 압박이 있으면 토론 3분을 1분으로 줄여도 됨 (*가장 인상적이었던 답 한 사람만 발표*)
