# 바이브코딩 (Vibe Coding)

> 정확한 문법이 아니라 "이런 느낌으로 만들어줘"라는 의도와 분위기로 코드를 만드는 새로운 코딩 방식.

## 1. 왜 "바이브"라고 부르나

- **누가**: Andrej Karpathy (전 OpenAI 창립 멤버, 전 Tesla AI 디렉터)
- **언제**: 2025년 2월 2일 — X(트위터) 게시물
- **왜 이 단어**:
  - 원문에서 그는 "fully give in to the **vibes**, embrace exponentials, and forget that the code even exists"라고 표현했다.
  - **vibe(바이브)**는 영어에서 "분위기·기분·느낌"을 뜻하는 일상어. 코드의 정확한 문법이 아니라 **만들고 싶은 분위기**가 출발점이라는 점을 강조.
  - 본인은 이 트윗을 "샤워하다 떠올라서 그냥 던진 잡생각(a shower of thoughts throwaway tweet)"이라고 회고했지만, 4,500만 회 이상 조회되며 정식 용어로 굳어졌다.

## 2. 그전에는 어떤 문제가 있었나

- **개발 언어를 따로 배워야 했다** — 영어/한국어가 아니라 Python·JavaScript 같은 별도 언어 학습이 진입 장벽.
- **세미콜론 하나, 괄호 하나 어긋나면 안 됐다** — 정확한 문법 없이는 한 줄도 실행되지 않았다.
- **머릿속 아이디어 → 실제 결과물**까지 가는 전달 과정이 너무 길었다. 기획자·디자이너가 떠올린 것을 개발자에게 설명하고, 개발자가 다시 코드로 옮기는 사이에 의도가 닳았다.
- **비개발자는 제작에 참여하기가 거의 불가능**했다.

**일상 비유**: 요리사에게 "매콤하면서도 깔끔한 느낌"이라고 말해도 알아듣기까지 오랜 학습이 필요했던 셈. 바이브코딩 이전에는 AI라는 요리사가 "정확한 그램 단위 레시피"만 받아들였다.

## 3. 어떻게 해결했나

- **자연어를 그대로 코드로 바꿀 수 있게** 됐다 — Cursor Composer + Claude Sonnet 같은 도구로 "이런 느낌으로 만들어줘" 한 마디면 코드가 나온다.
- **대화하듯 수정**하고, **빠르게 반복**하고, **즉시 실행**해본다.
- 결정적으로 **비개발자도 같은 자리에 앉을 수 있게** 됐다. 1강 메시지의 핵심.
- Karpathy 본인은 SuperWhisper로 음성 입력까지 결합해 "키보드를 거의 만지지 않는다"고 묘사.

**주의점**: 단, 바이브코딩은 모든 개발을 대체하지 않는다. Simon Willison 같은 비평가는 "프로덕션 환경에서는 한계가 있다"고 지적하며, AI가 만든 코드를 사람이 검수하지 않으면 위험하다는 의견도 함께 회자된다. (이 부분은 강의에서 "환각" 섹션과 연결)

## Before / After — 코딩 흐름의 변화

| 단계 | 옛날 (BEFORE) | 바이브코딩 (AFTER) |
|---|---|---|
| 의도 → 코드 | 기획서 → 명세 → 개발자 → 문법 정확한 코드 | "이런 느낌으로" → 즉시 코드 |
| 수정 | 코드 직접 편집, 다시 컴파일 | "이 부분만 더 부드럽게" → 자동 반영 |
| 비개발자 참여 | 거의 불가 | 디자이너·기획자도 직접 시연·수정 |
| 진입 장벽 | 6개월~수년 학습 | 한국어로 대화 가능 |

## 라이브 시연 가능한 예시

```bash
[강의 중 — 빈 폴더에서]
> "Pomodoro 타이머 만들어줘. 25분 일하고 5분 쉬는 거.
   디자인은 미니멀하게, 다크 테마로."

→ Claude Code: 1) HTML/CSS/JS 자동 생성
              2) 브라우저로 자동 띄움
              3) 작동 확인
              5분 안에 사용 가능한 결과물 완성
```
청중에게 *"오늘 한 번도 코드를 쓰지 않았는데 — 사용 가능한 도구 1개를 만들었다"* 는 충격을 직접 보여주는 가장 강력한 시연.

## 1년 변화 (2025-02 → 2026-05)

- **2025-02**: Karpathy 트윗으로 용어 등장. *"실험적·재미있는"* 활동.
- **2025-05**: 공식 사전(Merriam-Webster)이 *"watch list"* 등재.
- **2025-09**: 한국 IT 업계에서도 *"vibe coding"* 표기로 도입 사례 증가 (Toss·당근·라인 등 일부 팀 도입).
- **2026-03**: 비개발자 대상 *바이브코딩 부트캠프*가 한국에서도 다수 등장.
- **2026-05 (현재)**: *프로덕션* 코드는 여전히 사람 검토 필요. 그러나 *프로토타입·내부 도구·1회용 스크립트* 영역에서는 *"바이브가 표준"* 으로 정착.
