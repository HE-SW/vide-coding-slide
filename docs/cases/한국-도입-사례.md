# 한국 회사의 바이브코딩 / Claude Code 도입 사례

> 강의에서 *그대로 인용 가능*하도록 정리한 사례 모음. 모든 항목은 *외부 공개 자료* 기준이며 (회사 공식 블로그 / 보도자료 / 1차 인터뷰), 출처를 함께 표기. 2026-05-10 시점 검증.

## 1. 토스 (Toss)

### 핵심
- **언제부터**: 2025년 후반~2026년 초 시점, 사내 표준급으로 도입 확대
- **무엇을**: Claude Code를 핵심 도구로 채택. *"소프트웨어 3.0 시대"* 라는 자체 프레임으로 외부에 발표
- **누구**: 토스 엔지니어링 조직 전반. 디자인 시스템·코드 리뷰·구현 보조 영역에서 활용

### 공개 발언
> *"Cursor, Claude Code 같은 AI 기반 코딩 툴을 활용해 구현과 리뷰, 리팩토링을 처리하는 일이 국내외 프로덕트 조직에서 점점 일상화"* — 토스 기술 블로그 *"소프트웨어 3.0 시대를 맞이하며"*

### 강의 인용 톤
*"한국에서도 토스가 *소프트웨어 3.0 시대*라는 자체 슬로건으로 바이브코딩을 정식 사내 도구로 받아들였습니다. 토스 기술 블로그에 공식 글이 올라와 있어요."*

---

## 2. 당근 (Daangn)

### 핵심
- **무엇을**: *비개발자 구성원도* AI 코딩 도구(Cursor / Claude)로 사내 도구를 직접 만든다
- **운영 의식**: *AI Show & Tell* — 매주 화요일 각 팀이 AI 프로젝트와 시행착오를 사내에 공유
- **대표 사례**: 인턴(비개발자)이 *Claude와 함께 2주 만에* 회의 당번 봇 *"당번이"* 알파 버전 배포

### 공개 발언
> *"워드나 노션을 누구나 쓰듯이, AI 툴도 의지만 있다면 누구나 쓸 수 있다"* — 당근 기술 블로그 *"AI 툴 개발은 처음이라"*
>
> *"부동산팀 팀원들은 사전에 Cursor나 Claude 같은 AI 도구를 연습해 왔고, 당일에는 네 개의 그룹으로 나뉘어 각자 피처를 구현해냈다"*

### 강의 인용 톤
*"당근은 *비개발자 직원이 AI 도구로 사내 도구를 직접 만드는* 단계까지 왔습니다. 인턴이 Claude와 2주 만에 회의 봇 알파 버전을 배포한 사례가 외부 공개돼 있어요. 매주 화요일 'AI Show & Tell'이라는 사내 의식까지 만들어 운영합니다."*

---

## 3. 컬리 (Kurly)

### 핵심
- **무엇을**: Claude Code (v2.0.72, Opus 4.5) 사내 표준 도입. *"예측 가능한 바이브 코딩 전략"* 으로 공식 블로그 발행
- **팀 규모 / 효과**: OMS팀 4명이 *12개 MSA* 효율 운영. 단순 반복 작업이 줄어듦, 엔지니어는 *AI를 가르치는 선배 개발자* 역할
- **라이브 시연 문화**: 사내 발표에서 실제 Jira 티켓을 골라 *Frontend + Backend MSA를 동시에 라이브 바이브 코딩* 시연

### 공개 발언
> *"AI가 팀 코드 컨벤션을 완벽히 준수하여 코드를 작성하기 때문에, 엔지니어는 AI를 가르치는 선배 개발의 역할을 하게 됐다"* — 컬리 기술 블로그
>
> *"Claude Code를 활용한 예측 가능한 바이브 코딩 전략"* — 컬리 helloworld 블로그 시리즈 1편

### 강의 인용 톤
*"컬리는 4명짜리 작은 팀이 12개 마이크로서비스를 운영하는 사례를 외부에 공개했습니다. 핵심은 *예측 가능한 바이브 코딩* — 결과가 들쭉날쭉하지 않도록 *하니스를 깔아서* 운영하는 방식이에요. 컬리 기술 블로그에 두 편 시리즈로 올라와 있습니다."*

---

## 4. 1인 개발자 사례 — rhwp (edwardkim)

### 핵심
- **누구**: 한국 개발자 edwardkim
- **무엇**: *rhwp* — Rust로 만든 한국어 HWP 파일 뷰어/에디터 오픈소스 프로젝트
- **얼마나**: 50일간 *1인 + Claude Code + 외부 기여자 9명*으로 Rust 코드 100,000+ 라인, 941+ 테스트, 729개 문서 생산
- **단일 기록**: 2026-04-04 하루에 *기존 방식으로 2~3주 걸릴 작업*을 완료
- **방법론**: *하이퍼-워터폴(Hyper-Waterfall)* — 거시적으로는 워터폴, 미시적으로는 애자일

### 강의 인용 톤
*"가장 충격적인 1인 사례입니다. edwardkim이라는 한국 개발자가 50일 만에 Rust 코드 10만 라인 + 1천 개 가까운 테스트를 1인+AI로 만들어냈어요. 본인이 *하이퍼-워터폴*이라는 방법론까지 정립해서 외부에 공개했습니다."*

> 자세한 내용: `methodologies/하이퍼-워터폴.md`

## 보너스 — Anthropic 자사 워크플로우 (참고용 해외 사례)

해외 사례지만 한국 도입사들이 *벤치마크*하는 기준점이라 함께 묶음.

- Claude를 만든 회사 Anthropic이 *내부에서 Claude를 어떻게 쓰는가* 를 외부 발표 / 공식 블로그로 공개
- *코드 리뷰 자동화*, *법무·계약서 1차 검토*, *고객 지원 1차 응답* 등 다양한 부서에서 1차 도구로 활용

## 강의 운영 메모

- 사례별로 **1분 이내**로 끊어서 인용 (3개 사례 합쳐 5분이 적정)
- 청중 질문 *"한국 어디 회사가 진짜 쓰나요?"* 에 즉답 가능하도록 본 문서를 강의 전 한 번 훑어둘 것
- 각 회사 *공식 블로그 URL*은 강의 자료 첨부 파일로 함께 제공 (청중이 직접 검증 가능)
- 회사별 도입 *깊이·범위*는 외부 공개 정보 기준이며, 실제 사내 적용은 더 넓을 수 있음을 명시
