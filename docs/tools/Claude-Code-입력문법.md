# Claude Code 입력 문법 (Input Grammar) — `!` `@` `/` 와 키보드 단축키

> **2026-05-10 기준, Claude Code 2.x 계열.** 슬래시 명령어 *바깥*의 입력 트릭 모음. 프롬프트 첫 글자에 `!`·`@`·`/` 를 치는 것만으로 동작이 갈라지고, ESC를 한 번/두 번 누르는 것도 의미가 다르다. *명령어 표 어디에도 안 적혀 있는데 매일 쓰는* 것들.

## 1. 왜 별도의 "입력 문법"이 있나

Claude Code는 *대화창*이지만 동시에 *셸* 이기도 하다. 두 가지를 한 입력 줄에서 다 받기 위해, **첫 글자 prefix** 로 *어떤 처리 경로로 보낼지* 분기시킨다.

- `!ls` → 셸 명령어로 처리 (Claude 거치지 않음)
- `@src/auth.ts` → 파일 mention으로 처리 (자동완성·인용)
- `/help` → 슬래시 명령어 (사전 정의된 동작)
- `안녕` → 평범한 자연어 프롬프트

→ **첫 글자 한 글자가 라우팅을 바꾼다**. 이걸 모르면 셸 명령 한 번 돌리려고 *Claude에게 부탁*해서 수십 초·수백 토큰을 낭비한다.

## 2. 그전에는 어떤 문제가 있었나 — *모든 게 자연어 부탁*

prefix가 없던 초기 LLM 인터페이스(2023~2024 ChatGPT 등)는 *전부 자연어*. 셸 한 줄을 돌리려면 *"ls 명령어 결과 보여줘"* 라고 부탁 → 모델이 해석 → 도구 호출 → 결과. 모델이 *중간 의역*을 하니 안 정확하고, 토큰도 든다.

**일상 비유**: 식당에서 *모든 주문을 풀문장*으로 해야 한다고 상상해보자. *"제육 하나 부탁해요"* 대신 *"돼지고기와 양파, 고추장이 들어간 볶음 요리 1인분을 만들어주실 수 있을까요?"*. 메뉴판 번호로 *"3번 하나"* 하면 끝나는 걸. **prefix는 메뉴판 번호** 와 같은 단축 통로.

## 3. 어떻게 해결했나 — prefix 4개 + 키 단축키

## A. 입력 prefix — 첫 글자가 라우팅을 가른다

### `!` — 셸 즉시 실행

프롬프트 맨 앞에 `!`를 치면, 뒤의 명령은 **Claude를 거치지 않고 현재 터미널에서 직접 실행**된다. 출력은 자동으로 대화창에 흘러들어와 다음 턴의 *컨텍스트*가 된다.

```bash
> !npm test
[npm test가 직접 돌면서 결과가 실시간으로 흐른다]
✗ 3 tests failed in src/auth/login.test.ts

> 위 실패한 테스트 3개 고쳐줘
[Claude가 위에 흘러간 출력을 컨텍스트로 사용]
```

**언제 쓰나**: *Claude가 모르는 사실을 본인이 직접 확인*해야 할 때. 환경변수·디스크 사용량·`git status`·테스트 결과 같이 *지금 이 순간*에만 의미 있는 것들.

**Tab 자동완성**: 이전에 친 `!` 명령은 Tab으로 다시 불러올 수 있다.

> ⚠️ "현재 세션의 셸"이 아니라 *현재 터미널*에서 실행. cd로 디렉토리 옮기면 Claude의 작업 디렉토리도 같이 바뀜.

### `@` — 파일·디렉토리 mention

`@`를 치면 현재 작업 디렉토리 기준으로 파일 경로 자동완성이 뜬다. 골라 넣으면 *경로 토큰*으로 들어가고, **파일 내용은 Claude가 필요할 때 도구로 읽는다** (자동 인라인이 아님).

```bash
> @src/auth/login.ts 의 토큰 만료 처리 로직 설명해줘
[Claude가 Read 도구로 그 파일을 열어 답변]

> @docs/ 안의 모든 마크다운 한눈에 정리해줘
[디렉토리도 같은 식으로 mention 가능]
```

**왜 인라인이 아닌가**: 파일이 클 때 *전체 내용을 무조건 컨텍스트에 박는 것*은 토큰 낭비. mention은 *주소만 주고, Claude가 필요한 부분만 읽도록* 하는 설계.

### `/` — 슬래시 명령어

이미 잘 알려진 그것. `/clear`, `/compact`, `/plan` 등. 첫 글자 `/` 만 쳐도 사용 가능한 **빌트인 명령 + 사용자 Skill + 플러그인 명령** 이 모두 떠서 인터랙티브하게 고를 수 있다. 자세한 명령어 목록은 [Claude Code](Claude-Code.md).

### `#` — *더 이상 지원되지 않음* (2.x 기준)

1.x에선 `#` prefix가 *"이걸 CLAUDE.md에 추가할까요?"* 프롬프트를 띄웠다. **2.x에선 사라짐**. 같은 효과는 두 가지로 대체:

```bash
# 명시적 메모리 편집
> /memory                 # CLAUDE.md 편집기 띄움

# 대화로 부탁
> 이걸 프로젝트 CLAUDE.md에 규칙으로 추가해줘
```

또한 *auto memory* 시스템이 사용자 패턴을 자동 학습해 `~/.claude/projects/<project>/memory/MEMORY.md`에 저장하므로, 명시적 `#` 호출 빈도가 줄었다.

> 🔁 **강의 시 주의**: 옛 블로그·튜토리얼이 `#`를 알려주는 경우가 많다. *"이건 1.x 시절 문법, 지금은 `/memory`나 자동"* 이라고 정정해줄 것.

### `\` + Enter — 멀티라인 입력

`Shift+Enter`나 `Option+Enter`가 안 먹는 터미널에서도 동작하는 *모든 터미널 호환* 멀티라인 트릭. 줄 끝에 `\` 넣고 Enter.

## B. 키보드 단축키 — 한 번 vs 두 번

### `Esc` — 한 번과 두 번이 다른 일을 한다

| 횟수 | 동작 |
|---|---|
| 1회 | 현재 입력/생성 *취소*. 빈 프롬프트에서 누르면 세션 종료. |
| **2회 (빠르게)** | **리와인드 메뉴** — 이전 메시지로 *코드 변경까지 함께* 되감기 |

ESC 두 번은 사실상 `/rewind` 와 같다. **잘못된 방향으로 흘러가는 걸 알았을 때, 제일 먼저 누르는 키**. 청중에게 시연하면 *"AI가 망친 걸 한 번에 되돌릴 수 있구나"* 가 가장 강하게 와닿음.

### `Shift+Tab` — 권한 모드 순환

```text
default → acceptEdits → plan → (auto / bypassPermissions 활성화 시 추가)
```

매번 `/plan` 치고 다시 `/permissions` 치는 대신 한 키로 토글. 자세한 모드 의미는 [Claude Code 퍼미션 모드](Claude-Code-퍼미션모드.md).

### `Ctrl+R` — 프롬프트 히스토리 역방향 검색

bash의 `Ctrl+R`과 같은 결. *과거에 쳤던 프롬프트* 를 검색.

```text
Ctrl+R → 검색어 타이핑 → 매칭되는 과거 프롬프트가 미리보기로
Ctrl+R 반복 → 더 오래된 매칭으로 이동
Ctrl+S → 검색 범위 순환 (현재 세션 / 프로젝트 전체 / 모든 프로젝트)
Tab/Esc → 받아서 수정 가능
Enter → 즉시 실행
```

> ⚠️ Ctrl+R 은 *사고 토글이 아니다*. 사고/도구 출력 펼치기는 **`Ctrl+O`** (트랜스크립트 뷰어).

### `Option+T` (Mac) / `Alt+T` — 확장 사고 토글

*Extended thinking* (모델이 더 깊이 생각하는 모드) 켜기/끄기. v2.1.132+에서 macOS에선 별도 Meta 설정 없이 동작.

### 기타 자주 쓰는 키

| 키 | 동작 |
|---|---|
| `Tab` | 파일 경로 / `!` 명령 / 프롬프트 제안 자동완성 |
| `↑` / `↓` | 멀티라인 안에선 커서 이동, 끝에 도달하면 프롬프트 히스토리 |
| `Ctrl+C` | 현재 입력/생성 취소. 빈 프롬프트에선 EOF |
| `Ctrl+D` | 세션 종료 (EOF) |
| `Ctrl+L` | 화면 재그리기 (히스토리 보존) |
| `Ctrl+V` / `Cmd+V` / `Alt+V` | 클립보드 이미지 붙여넣기 — `[Image #N]` 칩으로 삽입 |
| `Shift+Enter` / `Option+Enter` | 멀티라인 줄바꿈 (`/terminal-setup` 후 활성) |
| `?` | 현재 환경의 단축키 도움말 표시 |

## Before / After — 같은 작업, 입력 문법 유무

```text
[BEFORE — prefix 없이 자연어로만]
> 현재 디렉토리에 어떤 파일들이 있는지 알려줘
[Claude가 LS 도구 호출 → 결과 해석 → 한국어로 정리해서 출력]
→ 30초·800 토큰

[AFTER — ! prefix 활용]
> !ls -la
[셸이 즉시 출력 → 그대로 컨텍스트에 박힘]
→ 0.2초·100 토큰
```

```text
[BEFORE — 파일 인용을 자연어로]
> src 폴더 안의 login.ts 라는 파일 좀 봐줘. 그게 src/auth 안에 있어
[Claude가 추측 → 잘못된 경로 시도 → 다시 시도]

[AFTER — @ mention]
> @src/auth/login.ts 의 토큰 처리 로직 설명해줘
[자동완성으로 정확한 경로 → 한 번에 도착]
```

## 라이브 시연 가능한 예시

### 시연 1 — `!`로 *지금 이 순간*을 박아넣기

```bash
> !date
Sat May 10 14:32:00 KST 2026

> 위 시각 기준으로 회의록 헤더 만들어줘
[Claude가 위 출력을 그대로 인용]
```
청중에게 *"AI가 시계가 없어도 셸로 박아주면 된다"* 가 즉시 전달됨.

### 시연 2 — ESC 두 번 되돌리기

```bash
> @src/users.ts 의 모든 변수명을 영어로 바꿔줘
[Claude가 50개 변수 일괄 변경]

[Esc Esc]
[리와인드 메뉴 등장 → 1단계 전 선택 → 변경이 통째로 되돌아감]

> 미안, 변수명만 말고 *영문 약어* 위주로 다시 해줘
```
*"AI가 망쳐도 한 번에 돌릴 수 있다"* — 강의에서 청중이 가장 안심하는 메시지.

### 시연 3 — `Ctrl+R`로 어제 명령 다시 호출

```bash
[Ctrl+R]
검색: tdd
> 어제 친 "tdd 사이클 한 번 돌려줘" 가 뜸
[Tab으로 받아서 살짝 수정 → Enter]
```

## 강의 연결 포인트

- **명령어 슬라이드** 자체보다 *"이 도구는 자연어만 받는 게 아니다"* 인상이 더 중요. `!`·`@`·ESC 두 번 — 이 셋만 시연해도 청중이 *터미널 도구로서의 Claude Code* 를 체감.
- **컨텍스트 절약 흐름** — `!` 와 `@` 는 *불필요한 모델 호출을 안 거치는* 가장 빠른 토큰 절약 수단. 비용·효율 이야기와 직접 연결.
