🤔 01 Think Before Coding
짐작하지 말고 드러내라
가정을 명시한다. 해석이 둘이면 둘 다 제시한다. 모호하면 멈추고 묻는다. → 잘못된 가정으로 달리는 사고 차단.
✂️ 02 Simplicity First
100줄로 될 일을 1,000줄로 짜지 마라
시키지 않은 기능·추상화·유연성·설정 가능성 금지. 200줄을 짰는데 50줄로 가능하면 다시 짠다.
🔪 03 Surgical Changes
부탁한 곳만 만져라
인접한 코드·주석·포맷 "개선" 금지. 망가지지 않은 걸 리팩터링 금지. 무관한 죽은 코드는 말만 하고 두기.
🎯 04 Goal-Driven Execution
검증 가능한 목표로 변환해라
"버그 고쳐줘" → "버그를 재현하는 테스트를 먼저 짜고, 통과시켜라". 강한 성공 기준이 AI를 스스로 루프하게 한다.
