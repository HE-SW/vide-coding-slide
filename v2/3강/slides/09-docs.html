docs/ — 프로젝트의 뇌

마크다운 3개로 AI한테 프로젝트 전체를 알려주기 — 각 문서가 한 가지 질문에 답한다

📘 PRD.md 뭘
→ 뭘 만드는지
# PRD: {프로젝트명} ## 목표 {한 줄 요약} ## 핵심 기능 1. {기능 1} 2. {기능 2} ## MVP 제외 사항 - {안 만들 것 1}
🏗️ ARCHITECTURE.md 어떻게
→ 어떻게 만드는지
# 아키텍처 ## 디렉토리 구조 {폴더 트리} ## 패턴 {디자인 패턴} ## 데이터 흐름 {데이터 흐름도}
🧭 ADR.md 왜
→ 왜 이렇게 만드는지
### ADR-001: {결정 사항} (예: Next.js App Router) **결정**: {뭘 선택} **이유**: {왜 선택} **트레이드오프**: {뭘 포기} ### ADR-002: {결정 사항} ...

세 문서가 합쳐져서 — AI가 "뭘·어떻게·왜"를 동시에 이해한 상태에서 답을 만듭니다.
새 멤버가 입사할 때 신입 교육 자료 한 묶음을 건네주는 것과 같은 일입니다.
