2026년 3월 — Claude Code 유출 사건

소스코드가 통째로 노출되자, 사람들이 그 안에서 발견한 것 '하니스 구조'

하니스 = AI를 혼자 두지 않고, 이걸 미리 정리해 둔 작업 환경
📖어떤 정보를
먼저 읽고
📐어떤 규칙을
참고하고
🧰어떤 도구를
쓸 수 있고
🔢어떤 순서로
행동할지
1
"절대 하면 안 되는 일" 목록 — 어떤 규칙을 참고하나
유출 소스: 메인 브랜치 강제 덮어쓰기 금지 · 되돌릴 수 없는 명령은 실행 전에 사용자에게 확인 · AI가 멋대로 저장(commit)·전송(push) 금지
👉 신입 사원 매뉴얼 첫 장의 "절대 하지 말 것" 목록
2
"명령 실행 전 23개 체크리스트" — 어떤 순서로 행동하나
유출 소스: 터미널 명령 하나를 실행하기 전에 23개 항목을 차례로 점검 ("시스템을 망가뜨리나 / 민감한 파일을 건드리나 …")
👉 비행기 이륙 전 조종사가 짚어 내려가는 점검표
3
"사용자가 짜증나 있는지" 감지 — 어떤 정보를 먼저 읽나
유출 소스: 욕설·짜증 표현 패턴이 통째로 박혀 있어서, 사용자가 "이거 왜 안 돼" 같은 말을 하면 AI가 톤을 바꾼다
👉 콜센터 상담 매뉴얼의 "고객이 화났을 때 응대법"
여러분이 Claude Code를 실행하는 그 순간, 이미 이 하니스는 작동하고 있었습니다.
