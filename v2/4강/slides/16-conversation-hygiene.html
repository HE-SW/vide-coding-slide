세션 길수록 폭증
Pattern #1 직격
— 한 패턴만 잡아도 비용 절반 가까이 줄어듭니다
예: 세션 초반에 "에러는 try/catch로 해" 했다고 해도, 50번 왕복 뒤 결과에 try/catch 빠져 있음. 그 잘못된 결과가 다시 컨텍스트에 들어가며 계속 망가짐.
1
/compact 선제 활용
컨텍스트 30~40% 넘기 전에 직접 친다. 자동 컴팩션은 늦음. 요약 가이드를 같이 줘서 중요한 부분이 안 잘리게.
/compact "디자인 결정과 주요 수정 파일만 남기고 정리"
2
한 세션 = 한 작업
"결제 끝났으니 OAuth도…" → 두 작업 정보로 컨텍스트 오염. 새 작업은 새 세션에서. 단 하나의 원칙.
// 끝났으면
/exit
// 새 세션에서 다음 작업
3
출력 형식 명시
기본 상태로 두면: 코드 5줄 바꾸고 30줄 설명 담는다. → 다 출력 토큰. CLAUDE.md에 박아두면 더 좋음.
