Meta가 보여준 두 가지 실패 패턴

"AI had no map"
Meta 데이터 파이프라인 (4 repos · 4,100 files) · engineering.fb.com
FAILURE 01
같은 값인데 이름이 다른 경우
결제 모듈은 price, 장바구니 모듈은 amount.
사람은 같은 값인 줄 알지만 코드엔 그 표시가 없다.
// payments.ts
price: number
// cart.ts
amount: number // 같은 값!
결과
컴파일 OK · 테스트 OK  →  프로덕션에서 가격 0 또는 단위 오류
(사일런트 버그)
FAILURE 02
deprecated인데 절대 못 지우는 값
멀티 repo가 sync하는 API enum.
한 repo에선 안 쓰지만 다른 repo가 의존한다.
// shared-types/order.ts
enum OrderStatus {
  // @deprecated
  LEGACY_PROCESSING
}
결과
결제 repo 정리 PR 머지  →  다음날 알림 · 분석 서비스 CI 깨짐
(4/5 repo 빌드 실패)
